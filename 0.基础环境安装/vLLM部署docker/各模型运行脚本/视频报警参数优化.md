单卡的核心瓶颈是显存。双卡时权重被 TP 切分到两卡，单卡要独自装下整个 27B 权重（AWQ-INT4 约 15GB），留给 KV cache 的空间从约 20GB 压缩到只剩 5~6GB。所以必须砍 --max-model-len（180K 是双卡才扛得住的），并发数和 CUDA graph也要跟着降。工具解析、多模态、fp8 KV cache 这些与硬件无关的保留不动。
单卡改造后命令：
  CUDA_VISIBLE_DEVICES=0 python -m vllm.entrypoints.openai.api_server \
    --model /data/models/Qwen3.6-27B-AWQ-INT4 \
    --served-model-name chaos3.6-27b \
    --host 0.0.0.0 \
    --port 8000 \
    --tensor-parallel-size 1 \
    --gpu-memory-utilization 0.92 \
    --max-model-len 32768 \
    --max-num-seqs 4 \
    --dtype float16 \
    --kv-cache-dtype fp8 \
    --enable-prefix-caching \
    --enable-chunked-prefill \
    --mm-processor-kwargs '{"min_pixels":200704,"max_pixels":1003520}' \
    --limit-mm-per-prompt '{"image":1}' \
    --cudagraph-capture-sizes 1 2 4 \
    --enable-auto-tool-choice \
    --tool-call-parser qwen3_coder \
    --reasoning-parser qwen3 \
    --default-chat-template-kwargs '{"enable_thinking": false}' \
    --trust-remote-code

  改动对比

  ┌───────────────────────────┬──────────────┬──────────────┬─────────────────────────┐
  │           参数            │ 原值（双卡） │ 新值（单卡） │          原因           │
  ├───────────────────────────┼──────────────┼──────────────┼─────────────────────────┤
  │ CUDA_VISIBLE_DEVICES      │ —            │ 0            │ 显式锁单卡，避免误占    │
  ├───────────────────────────┼──────────────┼──────────────┼─────────────────────────┤
  │ --tensor-parallel-size    │ 2            │ 1            │ 单卡不切分              │
  ├───────────────────────────┼──────────────┼──────────────┼─────────────────────────┤
  │ --max-model-len           │ 180000       │ 32768        │ 显存装不下，最大降幅项  │
  ├───────────────────────────┼──────────────┼──────────────┼─────────────────────────┤
  │ --max-num-seqs            │ 16           │ 4            │ KV cache 余量小，降并发 │
  ├───────────────────────────┼──────────────┼──────────────┼─────────────────────────┤
  │ --cudagraph-capture-sizes │ 1 2 4 8 16   │ 1 2 4        │ 必须 ≤ max-num-seqs     │
  └───────────────────────────┴──────────────┴──────────────┴─────────────────────────┘

  显存预算（单卡 24GB × 0.92 ≈ 22GB）： 权重 ~15GB + KV cache(fp8, 32K×4并发) ~5-6GB + CUDA/激活/graph ~1-2GB。
  启动前注意
  1. 先停掉当前双卡服务，否则 GPU0 显存被占满，单卡版起不来。
  2. 这只是基于 27B-AWQ 的估算值。如果启动时 OOM，按优先级降级：
    - --max-model-len → 24576 或 16384
    - 加 --enforce-eager（禁用 CUDA graph，再省 1~2GB，但小 batch 吞吐略降）
    - 多模态场景可降 max_pixels（图像 token 占 KV cache）
  3. 反之，若启动后 nvidia-smi 显示显存还有富余，可把 --max-num-seqs 提到 8、或 --max-model-len 提到 40960（fp8 下）。
